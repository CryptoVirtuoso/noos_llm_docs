# HouMo HLIELLama 部署使用指南

## 硬件环境

| 项目 | 详情 |
|------|------|
| 设备 | Moore Threads M1000 MQ50 Board |
| CPU | ARM Cortex-A78 × 12核，2.65 GHz |
| 系统内存 | 32 GB（CPU + GPU 共享） |
| GPU | Moore Threads iGPU，31.15 GB 统一内存 |
| **NPU** | **HoumoNPU LQ50，24 GB 专属内存，210 TOPS** |
| 存储 | 1TB NVMe SSD |
| OS | Ubuntu，Linux 6.6.10，aarch64 |

> NPU 的 24 GB 独立于系统 32 GB 之外，专供 HLIELLama 推理使用。

## model-cli 使用

### 命令总览

```bash
cd ~/houmo-HLIELLama-xh2

# 查询云端可用模型
./bin/model-cli query

# 按条件筛选
./bin/model-cli query "target=xh2"

# 下载模型（用 ID）
./bin/model-cli pull <ID>

# 查看本地已下载模型
./bin/model-cli list

# 查看当前加载中的模型
./bin/model-cli ps

# 加载模型（启动推理）
./bin/model-cli run <ID>

# 卸载模型
./bin/model-cli stop <ID>

# 删除本地模型
./bin/model-cli rm <ID>
```

## 启动推理服务（llama-server）

### 基本启动

```bash
cd ~/houmo-HLIELLama-xh2

./bin/llama-server \
  -m models/<模型文件名>.gguf \
  --port 17701 \
  --host 0.0.0.0 \
  -ngl 999
```

### 常用参数说明

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `-m` | 模型文件路径 | `models/xxx.gguf` |
| `-ngl N` | 卸载到 NPU 的层数（999=全部） | `999` |
| `-c N` | 上下文长度 | `32768` |
| `-t N` | CPU 线程数 | `8` |
| `-np N` | 并行槽位数（并发请求数） | `1` |
| `--port` | 监听端口 | `17701` |
| `--host` | 监听地址（0.0.0.0=外网可访问） | `127.0.0.1` |
| `--temp` | 生成温度 | `0.6` |
| `--top-p` | Top-P 采样 | `0.95` |
| `--ctx-size` | 同 `-c` | `32768` |

### Qwen3-30B-A3B 启动命令（开启思考模式）

```bash
cd ~/houmo-HLIELLama-xh2 && ./bin/llama-server \
  -m models/HiModel_xh2_qwen3_30b_a3b_256_32k_b1_1chip_2cores_v1.0.0_20260205.gguf \
  --port 17701 \
  --host 127.0.0.1 \
  -ngl 999 \
  -c 32768 \
  -t 8 \
  --temp 0.6 \
  --top-p 0.95 \
  --reasoning-format deepseek
```

### Qwen3-30B-A3B 启动命令（关闭思考模式）

使用 `bin/` 目录内置的非思考模板 `qwen3_nonthinking.jinja`，直接跳过 `<think>` 推理阶段，响应更快：

```bash
cd ~/houmo-HLIELLama-xh2 && ./bin/llama-server \
  -m models/HiModel_xh2_qwen3_30b_a3b_256_32k_b1_1chip_2cores_v1.0.0_20260205.gguf \
  --port 17701 \
  --host 127.0.0.1 \
  -ngl 999 \
  -c 32768 \
  -t 8 \
  --temp 0.7 \
  --top-p 0.95 \
  --repeat-penalty 1.15 \
  --repeat-last-n 256 \
  --chat-template-file bin/qwen3_nonthinking.jinja
```

> `qwen3_nonthinking.jinja` 位于 `~/houmo-HLIELLama-xh2/bin/`，HouMo 官方提供，专为关闭 Qwen3 思考模式设计。

### 关闭思考模式后的 API 字段说明

使用 `qwen3_nonthinking.jinja` 后，HouMo 版 llama.cpp 存在固定的字段路由行为：

| 字段 | 内容 | 说明 |
|------|------|------|
| `content` | 空 | HouMo 特有行为，无法改变 |
| `reasoning_content` | 实际答案 | 从此字段读取回复 |

**验证思考模式是否关闭：**
```bash
curl http://127.0.0.1:17701/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"1+1等于几"}],"max_tokens":200}' \
  | python3 -m json.tool | grep -E '"content"|"reasoning_content"'
```

- `reasoning_content` 有简短直接答案 → ✅ 思考已关闭
- `reasoning_content` 有大量推理过程 → ❌ 思考未关闭

**开发者 Python 调用示例：**
```python
msg = response.choices[0].message
# content 为空时读 reasoning_content
answer = msg.content or msg.reasoning_content
print(answer)
```

> WebUI（http://127.0.0.1:17701）聊天不受字段影响，显示正常。

### 多模态模型启动命令

#### 为什么有两个文件？

多模态（VLM）模型由两个独立模块组成：

| 文件 | 参数 | 作用 |
|------|------|------|
| `HiModel_xh2_qwen3-vl_*.gguf` | `-m` | **主模型**：负责语言理解、推理、生成回答 |
| `mmproj_xh2_qwen3-vl_*.gguf` | `--mmproj` | **视觉投影器**：把图片像素转成主模型能理解的向量 |

```
图片输入 → mmproj（眼睛：图片→向量） → 主模型（大脑：理解+回答）
```

纯文字模型（如 Qwen3-30B-A3B）只有一个文件，因为不需要视觉模块。

## NPU 监控

### 主命令：hm_smi（HouMo 专属，类似 nvidia-smi）

```bash
# 完整信息
hm_smi -a

# 实时刷新（每秒）
watch -n 1 'hm_smi -a'

# 只看关键指标
watch -n 1 'hm_smi -a | grep -E "IPU_Load|Core._Util|DDR_Memory|Core_Freq|Temperature"'
```

### hm_smi 输出解读

```
IPU_Load      : 100.0 %      ← NPU 整体负载
Core0_Util    : 100.0 %      ← 核心0利用率
Core1_Util    : 100.0 %      ← 核心1利用率
Core_Freq     : 1400.0 MHz   ← 当前运行频率（最高）
DDR_Memory_Total : 24448 MB  ← NPU 专属内存总量
DDR_Memory_Free  :  7052 MB  ← 剩余（跑30B模型后）
Core0 温度    : 42.3 C       ← 正常范围
Core1 温度    : 45.6 C       ← 正常范围
```

## 资源占用验证（Qwen3-30B-A3B 运行时）

### HLIELLama（NPU）资源占用

| 资源 | 数值 | 说明 |
|------|------|------|
| NPU 专属内存 | ~17 GB / 24 GB | 模型权重全部在 NPU 内存 |
| 系统 RAM（VmRSS） | **688 MB** | 极少，仅进程本身 |
| GPU 利用率 | 0% | GPU 完全不参与推理 |
| CPU 利用率 | 低（~13%） | 仅负责调度和 HTTP |
| 启动时间 | 9 秒 | |

> ✅ 确认：HLIELLama 仅使用 NPU 专属内存，不占用系统 RAM 和 GPU。

### 与 vLLM（GPU）对比

| 项目 | vLLM + MUSA（GPU） | HLIELLama（NPU） |
|------|-------------------|-----------------|
| 推理硬件 | Moore Threads iGPU | HoumoNPU LQ50 |
| 可用内存 | 31 GB（与系统共享） | **24 GB 专属** |
| 系统 RAM 占用 | **28.3 GB**（压垮系统） | **688 MB**（几乎不占） |
| 模型格式 | SafeTensors / GPTQ | GGUF（HouMo 专属编译） |
| 启动工具 | ExoBrain GUI | llama-server / model-cli |
| 端口 | 32102 | 17701 |
| Qwen3-30B 状态 | OOM 风险，内存紧张 | ✅ 充裕，稳定运行 |
| GPU 占用 | ✅ 使用（21%+ 底噪） | ❌ 不使用（0%） |
| NPU 占用 | ❌ 不使用 | ✅ 全部使用 |

### 两者可同时运行

由于资源完全隔离（GPU vs NPU），理论上可同时运行：
- vLLM（GPU，端口 32102）+ HLIELLama（NPU，端口 17701）
- 总内存：GPU 31 GB + NPU 24 GB = **55 GB 可用推理内存**

## 一、SDK 定位与核心概念

houmo-examples-xh2 是 HouMo（后摩智能）面向 XH2A NPU 的**底层开发 SDK**，提供从模型量化、编译到推理的完整工具链源码示例。

### 与 HLIELLama 2.0（lanyue2.0）的根本区别

| 维度 | HLIELLama 2.0（lanyue2.0） | houmo-examples v1.2.0 |
|------|---------------------------|----------------------|
| **定位** | 高层 LLM 推理框架（成品工具） | 底层 NPU 开发 SDK（源码示例） |
| **支持模型** | LLM + VLM（仅语言/多模态） | **全模态**：LLM / VLM / ASR / TTS / 检测 / 分割 / 分类 / OCR / Embedding / Reranker / Omni |
| **使用方式** | 二进制工具（llama-server） | Python 脚本 + C++ 源码 |
| **模型格式** | GGUF（内嵌 HMM） | HMM 原生格式 |
| **推理接口** | OpenAI 兼容 HTTP API | Python 直接调用 / C++ API |
| **量化工具** | model-cli 下载预量化模型 | ptq.py（自行量化） |
| **编译工具** | 无需（预编译） | build.py（编译为 HMM 格式） |
| **开发难度** | ★☆☆☆☆（直接用） | ★★★★☆（需懂量化/编译流程） |
| **适合人群** | 应用开发者 | AI 底层研发工程师 |
| **评测工具** | llama-bench | hmeval + llm_perf + tcim_perf |
| **特有能力** | Ollama 兼容层、model-cli | **TTS / ASR / 目标检测 / Embedding / Reranker** |

## 四、模型开发流程（每个模型通用）

每个模型目录下都有相同的文件结构，代表一套完整的 NPU 部署流程：

```
models/<category>/<model>/
├── get_model.py     # Step 1: 从 ModelScope/HuggingFace 下载原始模型
├── ptq.py           # Step 2: PTQ 后训练量化（INT8/INT4）
├── build.py         # Step 3: 编译为 HMM 格式（NPU可执行文件）
├── demo.py          # Step 4: 推理演示
├── config.yaml/.yml # 模型配置（量化参数、精度设置等）
├── requirements.txt # 依赖包
├── test.sh          # 一键测试脚本
└── README.MD        # 模型说明文档
```

### 完整工作流

```
原始模型（PyTorch/ONNX）
    ↓ get_model.py（下载）
原始权重文件
    ↓ ptq.py（量化）
量化校准模型
    ↓ build.py（hmatc编译）
.hmm 文件（NPU可执行）
    ↓ demo.py（推理）
推理结果
```

## 五、工具链详解

### 5.1 hmatc（HouMo ATC 编译器）

HouMo 专有的模型编译器，将量化后的模型编译为 XH2A NPU 可执行的 `.hmm` 格式。

```bash
# 安装
cd ~/houmo-examples-xh2/hmatc
bash install.sh
```

### 5.2 hmeval（评测框架）

Python CLI 工具，用于评测模型精度和性能。

```bash
# 安装
cd ~/houmo-examples-xh2/tools/hmeval
bash scripts/install.sh

# 使用
python -m hmeval --help
```

### 5.3 llm_perf（LLM 性能测试，C++）

专为 LLM 设计的 C++ 性能测试工具，支持：
- `HmllmInfer.cc` - 单批次 LLM 推理
- `HmllmInferMultiBatch.cc` - 多批次并行推理
- `HmvllmInfer.cc` - vLLM 兼容推理
- `HmEmbedding.cc` - Embedding 推理
- `device_monitor/` - NPU 设备监控
- `perf_tracker/` - 推理性能追踪（TTFT、TPOT、吞吐量）

```bash
cd ~/houmo-examples-xh2/tools/llm_perf
bash build_linux.sh
```

### 5.4 bandwidth_perf（带宽测试）

```bash
cd ~/houmo-examples-xh2/tools/bandwidth_perf
bash run.sh
```

### 5.5 computing_perf（算力测试）

```bash
cd ~/houmo-examples-xh2/tools/computing_perf
bash run.sh
```

### 5.6 hm_check（硬件检测，C++）

内置 3 个 `.hmm` 测试模型，验证 NPU 是否正常工作：
- `xh2_bandwidth_read.hmm` - 读带宽测试
- `xh2_bandwidth_write.hmm` - 写带宽测试  
- `xh2_compute.hmm` - 计算性能测试

```bash
cd ~/houmo-examples-xh2/tools/hm_check
bash build_linux.sh
./hm_check
```

## 八、与 ExoBrain 的关系和价值

### 当前 ExoBrain 架构

```
ExoBrain GUI（Tauri 2 + React）
    → Rust 后端
        → vLLM / HLIELLama（仅LLM推理）
```

### 基于 houmo-examples 的扩展架构

```
ExoBrain GUI（Tauri 2 + React）
    → Rust 后端
        ├── HLIELLama（LLM/VLM 推理）
        ├── ASR 服务（Whisper/SenseVoice → NPU）  ← 新增语音输入
        ├── TTS 服务（CosyVoice3 → NPU）          ← 新增语音输出
        ├── Embedding 服务（BGE/GTE → NPU）        ← 新增RAG能力
        ├── Reranker 服务（Qwen3 → NPU）           ← 新增RAG精排
        └── 视觉服务（YOLO → NPU）                  ← 新增图像感知
```

### 可构建的完整 AI 应用能力

| 能力模块 | 模型 | NPU负载 | 描述 |
|----------|------|---------|------|
| 对话推理 | Qwen3-30B-A3B（via HLIELLama） | ~17GB | 已有 |
| 多模态理解 | Qwen3-VL（via HLIELLama） | ~待测 | 已有 |
| 语音输入 ASR | Whisper/SenseVoice | 较小 | **新增** |
| 语音输出 TTS | CosyVoice3 | 较小 | **新增** |
| 知识库检索 | BGE/GTE Embedding | 很小 | **新增** |
| 检索精排 | Qwen3-Reranker | 较小 | **新增** |
| 图像识别 | YOLOv8m | 很小 | **新增** |
| OCR | GLM-OCR | 较小 | **新增** |

> **关键优势**：所有任务都在 NPU 24GB 专属内存中运行，不占用系统 32GB RAM，
> 理论上 ASR + TTS + Embedding + Reranker + YOLO 可以和 LLM **同时运行**

## 十、注意事项

1. **模型编译耗时**：ptq.py + build.py 可能需要数小时，仅需做一次
2. **存储需求**：原始模型 + 量化模型 + HMM，需要预留 2-3x 模型大小的空间
3. **与 HLIELLama 的互补关系**：
   - LLM 推理 → **用 HLIELLama**（已优化，有 OpenAI API）
   - ASR / TTS / CV / Embedding → **用 houmo-examples**
4. **hmatc 版本对齐**：确保 hmatc 编译器版本与驱动版本匹配（驱动 V1.0.0）
5. **NPU内存管理**：多个模型同时运行时注意 24GB 总量限制
